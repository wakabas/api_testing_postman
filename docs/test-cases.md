# Test-cases - JSONPlaceholder API

## TC - 01: Создание нового поста (позитивный)

**Предусловие**: API доступно
**Приоритет**: Высокий
**Тип**: Функциональное тестирование, Позитивное тестирование

**Шаги**:
1. Отправить POST запрос на {{baseUrl}}/posts
2. Тело запроса (JSON):
```json
 {
    "title": "Test post",
    "body": "banana",
    "userId": 1
}
```

**Ожидаемый результат**:

-Статус код 201

-Тело ответа содержит поле `id` с новым порядковым номером

-Поля title, body, userId совпадают с отправленными

**Реальный результат**: Соответствует

## TC - 02: Запрос к несуществующему эндпоинту (негативный)
**Предусловие**: API доступно
**Приоритет**: Средний
**Тип**: Функциональное тестирование, Негативное тестирование

**Шаги**:
1. Отправить GET запрос на {{baseUrl}}/postss (намеренная опечатка в пути)

**Ожидаемый результат**:

-Статус-код 404

-Пустое тело ответа

-Ответ содержит информацию об ошибке


**Реальный результат**: Соответствует
